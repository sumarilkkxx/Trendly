# Trendly 信息源优化方案

> 待人工确认后执行。如有不确定之处，请通过提问确认。

## 一、现状分析

### 1.1 当前数据流（scanner.js）

```
并行拉取：Hugging Face RSS | 自定义 RSS | Reddit | Twitter
    → 关键词匹配（若有）
    → OpenRouter AI 过滤（filterAndSummarize）
    → 入库
```

### 1.2 各源实现与依赖

| 源 | 文件 | 依赖/条件 | 当前状态 |
|----|------|----------|----------|
| Hugging Face | fetchRss.js | 无 | 固定 `huggingface.co/blog/feed.xml` |
| 自定义 RSS | fetchRss.js | 需用户在「消息源」添加 | 依赖用户配置 |
| Reddit | fetchReddit.js | REDDIT_CLIENT_ID/SECRET/REFRESH_TOKEN | 无配置则返回 [] |
| Twitter | fetchTwitter.js | TWITTERAPI_IO_API_KEY | 有 Key 则拉取 `queryType: Latest` |

### 1.3 问题根因

**问题 1：Twitter 低质内容多**

- 使用 `queryType: 'Latest'`，按时间排序，无互动过滤
- 不区分高互动与低互动推文，随手发的帖子也会被抓取
- TwitterAPI.io 返回 `likeCount`、`retweetCount` 等字段，**当前未使用**

**问题 2：来源单一**

- 若用户仅配置了 Twitter API Key，而 Reddit 未配置、自定义 RSS 未添加，则实际仅 Twitter 有数据
- 缺少零配置的「搜索引擎 / 新闻聚合」类内置源

## 二、优化方案（待确认）

### 2.1 针对问题 1：Twitter 质量提升 ✅ 已实现

| 方案 | 说明 | 状态 |
|------|------|------|
| **A. 改用 Top** | `queryType: 'Top'` 替代 `Latest`，按热度排序 | ✅ |
| **B. 互动阈值** | 点赞/转发/浏览三指标过滤，支持宽松/标准/严格/自定义 | ✅ |
| **C. 前端配置** | 设置页可切换模式并自定义阈值 | ✅ |

**已实现规则**：
- 宽松：点赞≥5，转发≥2，浏览≥200
- 标准：点赞≥10，转发≥5，浏览≥500
- 严格：点赞≥50，转发≥20，浏览≥2000（默认）
- 自定义：用户指定具体数值

---

### 2.2 针对问题 2：扩展信息来源

**新增内置源（零/低成本，无需额外 API Key）**：

| 源 | 类型 | 接入方式 | 说明 |
|----|------|----------|------|
| **Google News RSS** | 内置 | `https://news.google.com/rss/search?q={keyword}&hl=zh-CN&gl=CN&ceid=CN:zh-Hans` | 按关键词聚合新闻，免费 |
| **arXiv 学术** | 内置 | `https://rss.arxiv.org/rss/cs.AI`（或 cs.LG 等） | AI/ML 论文更新 |
| **Hacker News** | 内置 | Algolia `http://hn.algolia.com/api/v1/search?query=AI&tags=story&numericFilters=points>10` | 科技社区，按 points 过滤低质 |

**可选扩展（需 API Key）**：

| 源 | 说明 |
|----|------|
| Bing News API | 需 Azure 订阅，暂不纳入 |

**建议实现**：

1. 新增 `fetchGoogleNews.js`：根据关键词拼接 Google News RSS URL 拉取
2. 新增 `fetchArxiv.js`：拉取 cs.AI / cs.LG 等分类
3. 新增 `fetchHackerNews.js`：调用 Algolia API，`tags=story`，`numericFilters=points>10`（可配置）
4. 在 `scanner.js` 中并行调用上述新源

**数据格式**：统一为 `{ source, externalId, title, summary, url, rawContent, publishedAt }`，与现有逻辑兼容。

### 2.3 数据库与配置变更

| 变更 | 说明 |
|------|------|
| settings 新增 `twitter_min_engagement` | 默认 5，Twitter 互动阈值 |
| settings 新增 `hn_min_points` | 默认 10，HN 最低 points |
| settings 新增 `sources_google_news` | 1=启用，0=禁用 |
| settings 新增 `sources_arxiv` | 1=启用，0=禁用 |
| settings 新增 `sources_hackernews` | 1=启用，0=禁用 |
| 可选：`rss_sources` 表新增 `source_type` | 区分内置/自定义，便于前端展示 |

**前端**：在「通知设置」或「消息源」中增加上述开关与阈值配置（可后续迭代）。
